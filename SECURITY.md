# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in VidBack, please report it privately.

**Do not** create a public GitHub issue. Instead, email the details to **swahyuinfo@gmail.com**.

You can expect:
- An acknowledgement within **48 hours**
- A status update within **5 business days**
- Disclosure after a fix is deployed

## Scope

The following are in scope:
- Authentication bypass
- Data leaks or unauthorized access
- Cross-site scripting (XSS)
- SQL injection
- Remote code execution

The following are out of scope:
- Rate limiting issues
- Missing security headers
- Self-XSS
- Social engineering

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | ✅ Yes    |
| older   | ❌ No     |

## Responsible Disclosure

We ask that you:
- Give us reasonable time to fix the issue before public disclosure
- Make a good faith effort to avoid privacy violations
- Do not access or modify user data without permission
